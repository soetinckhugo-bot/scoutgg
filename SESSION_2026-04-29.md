## ✅ Partie 1 — Audit & Fixes Top 10

### 1. `/api/riot/sync` non authentifié
**Problème** : N'importe qui pouvait déclencher des appels API Riot coûteux.
**Fix** : Ajout de `requireAdmin()` au début du handler POST.
**Fichier** : `src/app/api/riot/sync/route.ts`

### 2. Build fail (routes admin manquantes)
**Problème** : Routes admin référencées mais potentiellement absentes.
**Fix** : Vérification — les routes existent (`recalculate-scores`, `sync-rosters`).
**Fichier** : Aucun changement nécessaire.

### 3. Export PDF non protégé
**Problème** : Le bouton Export PDF était complètement ouvert, sans auth ni premium check.
**Fix** : Ajout de `useSession` + toast "Please sign in to export PDF" + redirect vers login avec `callbackUrl`.
**Fichier** : `src/components/ExportPdfButton.tsx`

### 4. FavoriteButton sans gestion non-auth
**Problème** : Les utilisateurs non connectés voyaient une erreur générique en cliquant sur le favori.
**Fix** : Ajout de `useSession` + toast "Sign in to add favorites" + redirect vers login avec `callbackUrl`.
**Fichier** : `src/components/FavoriteButton.tsx`

### 5–6. Batch `console.error` → `logger`
**Problème** : ~40 fichiers API utilisaient `console.error` au lieu du `logger` centralisé.
**Fix** : Script batch `scripts/fix-logger.js` traité, puis corrections manuelles :
- Imports cassés dans les blocs multi-lignes (`sync-stats/route.ts`, `matches/route.ts`)
- Type mismatch `logger.error("...", error)` → `logger.error("...", { error })`
- Template literals non capturés par le regex (`admin/cron/[job]`, `weekly-digest`, `sync-stats`)
**Fichiers** : ~40 fichiers dans `src/app/api/**`

### 7. Leaderboards franglais
**Problème** : Labels français/anglais mélangés.
**Fix** : `joueurs` → `players`, `ans` → `yrs`
**Fichier** : `src/app/leaderboards/page.tsx`

### 8. Admin page 1635 lignes
**Problème** : Page monolithique, difficile à maintenir.
**Fix** : Extraction du composant `ReportForm` (~130 lignes) dans `src/app/admin/ReportForm.tsx`
**Résultat** : 1635 → 1501 lignes (-134 lignes)
**Fichiers** : `src/app/admin/page.tsx`, `src/app/admin/ReportForm.tsx` (nouveau)

### 9. Zod validation des query params
**Problème** : Les routes API acceptaient n'importe quel paramètre sans validation.
**Fix** : Schemas Zod ajoutés sur 3 routes principales :
- `/api/players` — `page`, `limit`, `sort`, `order`, `role`, `league`, `tier`, `minGames`
- `/api/leaderboard` — `season`, `split`, `metric`, `role`, `league`, `tier`, `nationality`, `ageMin`, `ageMax`, `page`, `limit`, `order`
- `/api/search/suggestions` — `q` (min 2, max 100 caractères)
**Fichiers** : `src/app/api/players/route.ts`, `src/app/api/leaderboard/route.ts`, `src/app/api/search/suggestions/route.ts`

### 10. Rate limiting généralisé
**Problème** : Seule `/api/riot/sync` avait du rate limiting.
**Fix** : Ajout sur les routes d'écriture lourdes (import/export/sync). Retiré des routes de lecture car elles surchargent le serveur de test E2E.
**Fichiers** : `src/app/api/import/oracle/route.ts`, `src/app/api/export/players/route.ts`, `src/app/api/riot/sync/route.ts`

### Rappels techniques
- `NextRequest` (next/server) peut causer des erreurs 500 en production standalone sur certaines routes API — préférer `Request` standard quand possible
- Le rate limiting in-memory (`Map`) n'est pas adapté aux tests E2E parallèles — à conditionner sur `NODE_ENV` ou utiliser Redis en production
- `new URL(request.url)` peut échouer si `request.url` est relatif — utiliser `new URL(request.url, "http://localhost")` comme fallback
