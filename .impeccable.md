# LeagueScout — Design Context

## Users

**Primary**: Esports scouts, team managers, and analysts in professional League of Legends organizations. They use LeagueScout to discover talent, compare players across leagues (LEC, LFL, LVP, Prime League, ROL, ERLs), and make data-driven recruitment decisions.

**Secondary**: Dedicated esports fans and aspiring analysts who want deep player performance insights beyond surface-level stats.

**Context**: Users spend extended sessions analyzing dense data — percentiles, radar charts, similarity scores, scouting reports. They need to quickly scan, compare, and extract actionable insights. The interface must support both rapid discovery (browsing) and deep analysis (player detail, side-by-side comparison).

## Brand Personality

**Three words**: Precise. Powerful. Professional.

- **Precise**: Every pixel, every data point, every interaction serves a purpose. No decoration without function.
- **Powerful**: The tool conveys authority and depth. Users feel equipped, not overwhelmed.
- **Professional**: Trustworthy, credible, institution-grade. Not a toy — a serious tool for serious decisions.

**Emotional goals**: Confidence ("I have the data I need"), clarity ("I understand this player instantly"), control ("I can filter, compare, and drill down effortlessly").

## Aesthetic Direction

**Visual tone**: Dark-first, data-dense, premium analytics. Think Bloomberg Terminal meets F1 telemetry — not a gaming site, a professional intelligence platform.

**Theme**: Dark mode default (`defaultTheme="dark"`). Light mode supported but secondary.

**Color philosophy**:
- Dominant: Deep navy/blacks (`#0F0F1A`, `#1A1A2E`) — authority, depth
- Accent: Coral-red (`#E94560`) — primary CTA, energy, urgency
- Data colors: Tier gradient (S+ cyan → C red) — semantic, instantly readable
- Neutrals: Tinted toward brand blue — never pure gray

**Typography**: Geist Sans (body, headings) + Geist Mono (data, monospace). Clean, modern, technical without being cold.

**Anti-references**:
- ❌ Dashboard Excel-like (too corporate, boring)
- ❌ Site de paris/skins (too flashy, cheap)
- ❌ Stats générique type op.gg/u.gg (pas différenciant)
- ❌ Gaming RGB/néons partout (immature)
- ❌ AI Slop: cyan-on-dark, purple gradients, glassmorphism, hero metric templates

## Design Principles

1. **Data first, chrome second**: The interface should get out of the way of the data. Every UI element must justify its existence.
2. **Density with clarity**: High information density is expected, but never at the cost of readability. Grouping, spacing, and hierarchy make density scannable.
3. **Consistency through tokens**: All colors, spacing, and typography must flow from design tokens — no hardcoded hex values, no arbitrary spacing.
4. **Motion with purpose**: Animations convey state changes and guide attention, never decorate. Prefer instant feedback over flashy transitions.
5. **Accessibility is non-negotiable**: WCAG 2.1 AA minimum. Keyboard navigation, screen reader support, and sufficient contrast are table stakes for a professional tool.
